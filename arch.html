Architecture Overview for Beginners & Content Authors

Layer 1 — Who uses AEM
✍️
Content Authors
Create, edit, and publish pages using drag-and-drop tools in the AEM Author environment.
🎨
Developers
Build templates, components, and custom functionality using Java, HTL, and OSGi.
⚙️
Administrators
Manage users, permissions, replication agents, and system configuration.
🌐
End Users / Visitors
Browse the final published website delivered via the AEM Publish tier.
↕ HTTP / Browser
Layer 2 — Presentation & Delivery
🖥️
Dispatcher (CDN Cache)
Acts as a reverse proxy & caching layer. Delivers static pages fast and protects the Publish server.
📡
CDN / Load Balancer
Distributes global traffic, reduces latency, and improves performance for visitors worldwide.
📱
Channels
Websites, mobile apps, email, IoT, digital signage — AEM delivers content to any channel.
↕ Sling URL Routing
Layer 3 — AEM Core Platform ★ Heart of AEM
✏️ Author Environment
  • 1 Author logs into AEM Author UI (Sites, Assets, Forms)
  • 2 Creates & edits pages using editable templates & core components
  • 3 Uses Workflows for review, approval & translation
  • 4 Activates (publishes) content to the Publish tier
🚀 Publish Environment
  • 1 Receives replicated content from Author
  • 2 Renders pages dynamically via Sling & HTL templates
  • 3 Serves requests through Dispatcher cache
  • 4 Scales horizontally with multiple Publish instances
🔩
OSGi Framework
Modular Java runtime powering all AEM services & bundles (Apache Felix).
🔀
Apache Sling
Maps URLs to content nodes in the JCR repository. The rendering engine of AEM.
📐
HTL / Templates
HTML Template Language — AEM's secure, server-side templating system used by developers.
🔁
Replication
Syncs activated content from Author to Publish instances automatically.
↕ JCR / Oak Storage API
Layer 4 — Content Repository
🌳
JCR (Java Content Repository)
The central tree-structured database where all content, assets, and config are stored as nodes.
🪨
Apache Jackrabbit Oak
The underlying JCR implementation. Supports MongoDB or Segment Node Store for storage.
🖼️
DAM (Digital Asset Mgmt)
Stores images, videos, PDFs. Authors pick assets from DAM to place on pages.
🔍
Apache Lucene / Oak Index
Full-text search indexing for content queries and fast content discovery.
↕ APIs & Connectors
Layer 5 — Integrations & Adobe Ecosystem
📊
Adobe Analytics
Tracks visitor behaviour. Authors can tag pages with analytics events.
🎯
Adobe Target
Personalization & A/B testing. Deliver different content to different audiences.
🔗
Adobe Launch / Tags
Tag management system — deploy marketing scripts without developer involvement.
💬
Third-Party APIs
CRM, e-commerce, translation services, social media, and more via REST/GraphQL.
↕ OS / Cloud Infrastructure
Layer 6 — Infrastructure
☁️
AEM as a Cloud Service
Adobe-managed, auto-scaling cloud deployment. No manual patching. Latest AEM always.
🏢
On-Premise / AMS
Self-hosted on own servers or Adobe Managed Services. Full infrastructure control.
🐳
Containers / Kubernetes
AEM Cloud uses Docker containers orchestrated by Kubernetes for resilience & scale.
